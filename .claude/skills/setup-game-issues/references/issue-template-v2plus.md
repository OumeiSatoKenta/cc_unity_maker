## テンプレート記入ルール

1. **全セクション必須** — プレースホルダーのまま残さない
2. **難易度設計**: パラメータ変更だけでなく、各ステージに「新ルール/要素」を必ず追加する
3. **ゲームデザインパターン**: 6種から最低1つ選択し、ゲーム固有の適用方法を記述する
4. **判断ポイント**: 「タップする」だけでは不十分。「どこを」「いつ」「何を犠牲にして」を具体的に記述する
5. **チュートリアル情報**: InstructionPanel用。1行で分かる簡潔な表現にする
6. **スコア**: コンボ/乗算/連鎖ボーナスなどの深みのあるスコアシステムを設計する

## 削除すべき旧セクション

以下のセクションが既存Issueに含まれている場合は削除する:
- `必要な GameObject 一覧`（プレースホルダーのみ）
- `Unity 実装上の注意`（プレースホルダーのみの場合。実際に有用な内容がある場合は「ゲーム概要」等に統合）
- `Claude Code への依頼文（コピペ用）`
- `Unity 実装方針`（プレースホルダーのみの場合）
