# Unity コンパイル検証スキル

Unity MCP 経由でコンパイルを強制実行し、エラーを確認・自動修正するスキル。

**引数:** ゲームID（省略可。指定するとエラー修正時の対象ファイルを絞り込む）

---

## 実行手順

### 0. Unity MCP セッション確認（必須・最初に実行）

`mcp__coplaydev-mcp__manage_editor(action="status")` を呼び出す。

- **エラーが返った場合 → 即座に終了する:**
  ```
  ⚠️ Unity MCP セッションなし: コンパイル検証をスキップします。
  ```
  以降のステップは**一切実行しない**。

- **正常に応答が返った場合 → ステップ1へ進む。**

### 1. コンパイル実行

`mcp__coplaydev-mcp__refresh_unity` ツールを使用する:
```
mcp__coplaydev-mcp__refresh_unity(mode="force", scope="all", compile="request", wait_for_ready=true)
```

### 2. エラー確認

`mcp__coplaydev-mcp__read_console` ツールを使用する:
```
mcp__coplaydev-mcp__read_console(types=["error"], count=20)
```

### 3. 判定と対応

**エラーがない場合:**
- 「✅ コンパイル成功: エラーなし」と報告して完了。

**エラーがある場合:**
- エラーメッセージを解析し、対象ファイルを特定する。
- スクリプトを修正する。
- 手順 1〜2 を繰り返す（最大5回）。
- 5回試みても解消しない場合は、残っているエラーをリストアップしてユーザーに報告する。

### 4. 結果レポート

以下の形式で報告する:

```
## コンパイル検証結果
- 状態: ✅ 成功 / ❌ 失敗
- エラー件数: N件
- 修正したファイル: （あれば列挙）
```

---

## MCP 接続がない場合

ステップ0で検出済みのため、このセクションは参照不要。
