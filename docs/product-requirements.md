# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**cc_unity_maker** - 自然言語でUnityゲームを量産するAI駆動ワークフローシステム

### プロダクトコンセプト
- **ゼロコード実装**: 非エンジニアがClaude Codeに話しかけるだけで、UnityゲームのC#スクリプト・シーン構成・プロジェクト一式が自動生成される
- **ゲームアイデアの体系的消化**: cc_game_ideasの100本のアイデアをGitHub Projects（スプレッドシート形式）で管理しながら順次実装する
- **Unityを開くだけでプレイできる**: Claude Codeが生成したプロジェクトはUnity Editorで開いてPlayボタンを押せば即プレイ可能な状態を目指す

### プロダクトビジョン
プログラミングができない社員でも、Claude Codeに「このゲームを作って」と伝えるだけで
カジュアルゲームを量産できる仕組みを作る。
アイデアの選択からUnityプロジェクトのGitHub push完了まで、
非エンジニアが迷わず進められるガイド付きワークフローを提供する。

### 目的
- 非エンジニアが1日1本（工数S）ペースでUnityゲームを完成させる
- コードを書かずにゲームが作れる体験を社内に広める
- Claude CodeとUnityの組み合わせによるゲーム量産ノウハウを蓄積する

---

## ターゲットユーザー

### プライマリーペルソナ: 社内の非エンジニアゲーム制作担当者
- プログラミング経験なし（またはごく初歩的）
- Unity Editorの基本操作（開く・Playボタン押す・動作確認）は習得済み、またはこれから習得
- GitHubアカウントあり、基本的なGit操作（クローン・push）はガイドがあればできる
- Claude Codeを使った経験は浅い（または初めて）
- ゲームのアイデアを出すことは得意

**現在の課題**:
- ゲームを作りたいがコードが書けない
- Unity操作を覚えながらゲーム設計も考えるのは負荷が高い
- 「何から始めればいいか」のフローが不明確

**期待する解決策**:
- Claude Codeに話しかけるだけでUnityプロジェクトが出来上がる
- 「次にやること」が常に明確なステップガイドがある
- GitHub Projectsで今どのゲームを作っているか一目でわかる

## 成功指標 (KPI)

### プライマリーKPI
- **完成ゲーム数**: 工数Sのゲームを月5本以上完成させる（3ヶ月後）
- **操作完結率**: ユーザーがエラーで詰まらずに1ゲームを完成できる割合 80%以上

### セカンダリーKPI
- **1ゲームあたりの所要時間**: アイデア選択からUnity動作確認まで2時間以内（工数S）
- **Claude Codeへの依頼回数**: 1ゲームあたり5回以内の指示でプロジェクト完成
- **GitHub Projects更新率**: 完成したゲームは1営業日以内にステータスが更新される

## 機能要件

### コア機能（MVP）

#### 1. ゲーム仕様展開機能

**ユーザーストーリー**:
非エンジニアとして、ゲームIDとタイトルを伝えるだけで実装可能な仕様書を得るために、
Claude Codeによる自動仕様展開が欲しい

**受け入れ条件**:
- [ ] 「ゲーム001 BlockFlowを作って」の一言で仕様展開が開始できる
- [ ] 生成仕様に以下が含まれる:
  - ゲーム概要（100字以内の非エンジニアが読んでわかる説明）
  - コアループ（プレイヤーが繰り返す基本行動を3ステップ以内で説明）
  - 必要なGameObjectと役割の一覧
  - スコア/クリア条件
  - Unity実装方針（使用コンポーネント・物理/UI/Inputの選択）
- [ ] 工数見積もりの妥当性が検証され、必要なら修正される

**優先度**: P0（必須）

---

#### 2. Unityプロジェクト自動生成機能

**ユーザーストーリー**:
非エンジニアとして、C#を書かずにUnityで動作するゲームを得るために、
Claude Codeがプロジェクト一式を自動生成する機能が欲しい

**受け入れ条件**:
- [ ] `projects/<ID>-<タイトル>/` 配下にUnityプロジェクト構造が生成される
- [ ] 以下のファイルがClaude Codeによって自動生成される:
  - `Assets/Scripts/` 配下のC#スクリプト（コアメカニクス・UI・GameManager）
  - `Assets/Scenes/` 配下のシーンファイル（Editorスクリプトによる自動構成）
  - `README.md`（Unity Editorでの起動手順・操作方法）
- [ ] Unity 6以降のAPIに準拠し、コンパイルエラーが発生しない
- [ ] Playボタンを押すだけでゲームが起動する状態になっている
- [ ] ユーザーはUnity Editorを開いてPlayを押すだけでよい

**優先度**: P0（必須）

---

#### 3. GitHub Projects連携機能

**ユーザーストーリー**:
非エンジニアとして、100本のゲームの進捗を把握するために、
スプレッドシート感覚で使えるGitHub Projectsで管理したい

**受け入れ条件**:
- [ ] GitHub Projectsのテーブルビューに全100本のゲームが登録されている
- [ ] 各行に以下の列がある: ID・タイトル・カテゴリ・工数・ステータス・担当者・Issueリンク・完成日
- [ ] ステータスは `未着手` / `作業中` / `完成` の3種類でドロップダウン操作可能
- [ ] フィルター機能で工数・カテゴリ・ステータスで絞り込みできる
- [ ] 各ゲームのIssue（実装仕様）へのリンクがProjectから1クリックで開ける

**優先度**: P0（必須）

---

#### 4. ゲーム実装Issueテンプレート

**ユーザーストーリー**:
非エンジニアとして、Issueを見れば次に何をすればよいかわかるために、
ステップバイステップの実装チェックリスト付きIssueが欲しい

**受け入れ条件**:
- [ ] Issueに以下のセクションが含まれる:
  - ゲーム概要（非エンジニアが読んでわかる説明）
  - 実装仕様（オブジェクト・スクリプト・シーン構成）
  - Claude Codeへの依頼文テンプレート（コピペで使える）
  - 実装チェックリスト（順序付きで何をすべきか明確）
  - 動作確認チェックリスト
- [ ] ラベルにカテゴリ（puzzle/action/casual等）と工数（size-S/M/L）が付与される

**優先度**: P0（必須）

---

#### 5. 非エンジニア向けスタートガイド

**ユーザーストーリー**:
Claude Code・Unity・GitHubを使い始めたばかりの社員として、
何も知らない状態から最初のゲームを完成させるために、
迷わず進められるセットアップガイドが欲しい

**受け入れ条件**:
- [ ] 以下のセットアップ手順が `docs/GETTING_STARTED.md` にある:
  - GitHubアカウント作成・リポジトリのクローン
  - Unity Hub・Unity 6のインストール
  - Claude Codeのインストールと起動
  - 最初のゲームを作るまでのStep-by-step（スクリーンショット付き）
- [ ] ガイド通りに進めば最短2時間で1本目のゲームが完成できる

**優先度**: P0（必須）

## 標準ワークフロー

```
【Step 1】 次のゲームを選ぶ
  GitHub Projects のテーブルビューを開く
  → ステータス「未着手」× 工数「S」でフィルター
  → 気になるタイトルを1つ選んでステータスを「作業中」に変更

【Step 2】 Issueを確認する
  ProjectからIssueリンクをクリック
  → ゲーム概要・実装仕様・Claude Codeへの依頼文を確認

【Step 3】 Claude Codeに依頼する（ターミナルで）
  > 「ゲーム001 BlockFlowを作って」
  → Claude Codeが仕様確認 → プロジェクト生成 → GitHubにpush

【Step 4】 Unityで動作確認する
  Unity Hubで projects/001-BlockFlow を開く
  → Playボタンを押してゲームが動くか確認
  → バグや改善があればClaude Codeに伝える
    例:「ブロックが消えない」「もっと速くしたい」

【Step 5】 完成・ステータス更新
  GitHub ProjectsでステータスをDoneに変更
  → Issueをクローズ
  → 次のゲームへ
```

## 非機能要件

### パフォーマンス
- Claude Codeによるプロジェクト生成時間: 工数Sのゲームで10分以内
- GitHub Projectsの表示速度: 100件のデータでフィルター結果が3秒以内

### ユーザビリティ
- セットアップガイドを読めば、プログラミング未経験者が2時間以内に1本目を完成できる
- Claude Codeへの依頼は日本語の自然文で行える
- エラーが発生した場合、Claude Codeが原因と解決策を日本語で説明する

### 信頼性
- 生成されたC#コードはUnity 6でコンパイルエラーが発生しない
- プロジェクト生成後、Playボタンで必ずゲームが起動する（クラッシュしない）

### 保守性
- 新しいゲームジャンルへの対応は、プロンプトの追加のみで可能
- ワークフロー手順はCLAUDE.mdとsteeringスキルで管理

---

## スコープ外

- ゲームのサウンドデザイン・BGM制作
- App Store / Google Play へのリリース
- マルチプレイヤー対応ゲームの実装
- 詳細な3Dモデリング（プリミティブ形状・無料アセット利用に限定）
- ゲームのマーケティング・収益化
- Unity以外のゲームエンジン（Godot等）への対応
