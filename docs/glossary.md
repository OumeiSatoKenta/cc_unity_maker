# 用語集 (Glossary)

このプロジェクトで使用する用語を統一定義します。
ドキュメント・Issue・Claude Codeへの依頼文では、この用語集の表記を使用してください。

---

## プロジェクト固有用語

### cc_unity_maker
このリポジトリ全体を指す。ゲームアイデアをUnityゲームに変換するワークフローシステム。

### cc_game_ideas
ゲームアイデア100本を管理する別リポジトリ（`https://github.com/OumeiSatoKenta/cc_game_ideas`）。
`ideas/ideas.md` にアイデア一覧が記載されている。

### ゲームアイデア / GameIdea
cc_game_ideas の `ideas.md` に定義された1行のゲーム企画。ID・タイトル・コアメカニクス・工数・ステータスで構成される。

### ゲーム実装仕様 / GameSpec
Claude Code がゲームアイデアを展開して生成する詳細仕様書。GitHub Issue の本文として管理される。
ゲーム概要・コアループ・シーン構成・必要なGameObject・実装チェックリストを含む。

### ミニゲーム集
全100ゲームを1つのUnityプロジェクトにまとめたもの。TopMenuでゲームを選んでプレイする形式。

## ワークフロー用語

### 仕様展開
ゲームアイデア（1行）からゲーム実装仕様（詳細設計）を生成するClaude Codeの処理。
「ゲーム001 BlockFlow を作って」と依頼することで開始される。

### SceneSetup
Unity Editorのメニュー（Assets > Setup > \<タイトル\>）から実行するEditorスクリプト。
シーンへのGameObject配置・コンポーネント設定・スクリプトのアタッチを自動化し、
非エンジニアがインスペクタを手動操作する必要をなくす。

### GameRegistry
`unity/Assets/Resources/GameRegistry.json` に配置されるゲーム一覧データ。
TopMenuがこのファイルを読み込んでゲームカードを表示する。
新ゲーム追加のたびにClaude Codeが自動更新する。

### TopMenu
アプリ起動時に表示されるゲーム選択画面。カテゴリ別タブでゲームカードを一覧表示し、タップするとゲームシーンへ遷移する。

### ゲームカード
TopMenuでゲームを選ぶためのUI部品。ゲームID・タイトル・工数を表示する。未実装のゲームはグレーアウト。

---

## 進捗管理用語

### GitHub Projects（テーブルビュー）
全100ゲームの進捗をスプレッドシート感覚で管理するGitHub機能。
列: ID・タイトル・カテゴリ・工数・ステータス・担当者・Issueリンク・完成日。

### ステータス
ゲームの進捗状態。以下の3種類のみ使用する:
- **未着手**: まだ実装を開始していない
- **作業中**: 現在実装中（または実装予定で着手済み）
- **完成**: Unity で動作確認済み、Issueクローズ済み

### 工数
ゲームの実装難易度の目安:
- **S**: 1日程度（ワンメカニクスのシンプルなゲーム）
- **M**: 1週間程度（複数画面・スコア・セーブがあるゲーム）
- **L**: 2週間程度（育成要素・ストーリー・多機能なゲーム）

## Unityプロジェクト用語

### 単一プロジェクト方式
全ゲームを1つのUnityプロジェクト（`unity/`）で管理する方式。
新ゲーム追加 = シーンとスクリプトの追加のみ。プロジェクトは作り直さない。

### シーン / Scene
Unityのゲーム画面の単位。TopMenuシーンと各ゲームシーンで構成される。
命名規則: `<3桁ID>_<PascalCaseタイトル>.unity`（例: `001_BlockFlow.unity`）

### コアメカニクス
そのゲームの中心となる操作・ルール。ゲームアイデアの「コアメカニクス」列に記載される1行の説明。
例: 「色付きブロックをスワイプして同色を全て繋げる」

### コアループ
プレイヤーが繰り返す基本行動のサイクル（3ステップ以内で定義）。
例: ① スワイプでブロックを動かす → ② 同色が繋がると消える → ③ 全て消したらクリア

### BackToMenuButton
全ゲームシーンに配置する共通コンポーネント。タップするとTopMenuシーンに戻る。
`Common/BackToMenuButton.cs` として定義される。

### 名前空間 / Namespace
C#でスクリプトの衝突を防ぐための識別子。ゲームごとに `namespace Game<ID>_<Title>` を付与する。
例: `namespace Game001_BlockFlow`

## ツール用語

### Claude Code (CC)
Anthropicが提供するAI駆動の開発アシスタント。ターミナルから起動し、自然言語での指示でコード生成・ファイル操作・Git操作を自動実行する。このプロジェクトではゲームの仕様展開・Unityプロジェクト生成・GitHub Issue作成を担当する。

### gh CLI
GitHubの公式CLIツール。Claude Codeから呼び出してIssue作成・Projects更新・PR作成を行う。

### Unity Hub
Unityのバージョン管理・プロジェクト管理ツール。プロジェクトを開く際はUnity Hub経由で行う。

### Unity Editor
ゲーム開発のメインツール。SceneSetupスクリプトの実行とPlayボタンによる動作確認に使用する。

### Inspector（インスペクタ）
Unity EditorでGameObjectのコンポーネント設定を確認・変更するパネル。
このプロジェクトでは非エンジニアがインスペクタを手動操作する場面をSceneSetupで極力ゼロにする。

### Play ボタン
Unity Editorでゲームを実行するボタン（▶）。押すとシーンがゲームとして動作する。
動作確認の最終ステップ。

### Tooltip
`[SerializeField]` に付与するUnityの注釈。インスペクタでマウスオーバーすると説明が表示される。
非エンジニアがフィールドの意味を把握するために全SerializeFieldに付与する。

## Git / GitHub 用語

### feature/game-\<ID\>-\<title\> ブランチ
各ゲームの実装ブランチ。1ゲーム = 1ブランチ = 1PR を基本とする。
例: `feature/game-001-block-flow`

### Issue（ゲーム実装仕様書）
各ゲームの実装仕様を記載するGitHub Issue。タイトル・仕様・Claude Codeへの依頼文テンプレート・実装チェックリスト・動作確認チェックリストを含む。

### implemented フラグ
`GameRegistry.json` の各ゲームエントリに付与するboolean値。
`true`: 実装済み（TopMenuでタップ可能）、`false`: 未実装（グレーアウト）。
