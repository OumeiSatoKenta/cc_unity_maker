## 削除対象セクション

以下は全件から削除:
- `必要な GameObject 一覧`（プレースホルダーのみ）
- `Unity 実装上の注意`（プレースホルダーのみのもの。実際に内容があるものは残す）
- `Claude Code への依頼文（コピペ用）`

## 処理フロー

```
1. 全100件のIssue bodyを一括取得
2. 10件ずつのバッチに分割
3. 各バッチをサブエージェントで並列処理:
   a. 既存bodyを解析
   b. 不足セクションを生成（ゲーム概要から推論）
   c. v2+テンプレートに組み立て
   d. gh issue edit で更新
4. 結果を集約・確認
```

## レート制限対策

- GitHub API: 1秒間隔でリクエスト
- バッチ間に3秒の待機
- 失敗時はリトライ（最大3回）

## 実装の順序

1. 全Issue bodyの一括取得とJSON保存
2. v2+テンプレートスクリプトの作成
3. バッチ1: Issue 001-010（パイロット、手動確認）
4. バッチ2-10: Issue 011-100（並列実行）
5. 全件の更新確認
