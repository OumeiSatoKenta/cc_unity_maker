# 要求内容

## 概要

既存の100件のゲームIssue（001〜100）を v2+ 拡張フォーマットに一括更新する。

## 背景

- v1形式（001〜032）: 操作仕様テーブル・UI要素・スコア基準なし、プレースホルダーだらけ
- v2形式（033〜100）: 操作仕様テーブル・クリア条件・UI要素あり、ただし難易度設計なし
- v2+形式（101 ChainReactor）: 5ステージ難易度設計・ゲームデザインパターン・判断ポイント対応

add-feature v2 改善により、実装時に以下が必須になった:
- 5ステージ進行（StageManager統合）
- チュートリアル（InstructionPanel）
- ゲームデザインパターン（6種カタログから選択）
- コンボ/乗算スコア

Issueがこれに対応していないため、実装時に仕様が不足する。

## 実装対象

### 1. 全100件のIssueフォーマット統一（v2+拡張）

追加セクション:
- **難易度設計（5ステージ）**: 各ステージのパラメータ変化と新ルール追加の設計
- **ゲームデザインパターン**: 6種カタログから該当パターンを選択
- **判断ポイント**: プレイヤーが意思決定する具体的場面を3つ以上
- **チュートリアル情報**: InstructionPanel用の4テキスト（title/description/controls/goal）

### 2. v1形式（001〜032）の底上げ

v1にしかない不備を修正:
- 操作仕様・ゲームフローテーブルの追加
- クリア/ゲームオーバー条件の具体化
- スコア・評価基準の追加
- UI要素一覧の追加

### 3. 不要セクションの削除

全件から削除:
- `必要な GameObject 一覧`（プレースホルダーのみで無価値）
- `Unity 実装上の注意`（プレースホルダーのみのもの）
- `Claude Code への依頼文（コピペ用）`（不要）

## 受け入れ条件

- [ ] 全100件のIssueが統一フォーマットに更新されている
- [ ] 各Issueに難易度設計（5ステージ）セクションがある
- [ ] 各Issueにゲームデザインパターンが選択されている
- [ ] 各Issueに判断ポイントが3つ以上記載されている
- [ ] 各Issueにチュートリアル情報（4テキスト）がある
- [ ] v1形式の不備（操作仕様・クリア条件等）が修正されている

## スコープ外

- Issue のタイトルやラベルの変更
- 新規Issueの作成
- 実装コードの変更
